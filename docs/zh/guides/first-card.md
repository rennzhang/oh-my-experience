# 第一张经验卡

OME 真正有价值，是因为经验库里有你自己的真实纠正。第一张经验也应该走复盘和草稿审批：
Agent 复盘，你确认和补充，确认后再入库。

## 1. 开始复盘

把这段话复制给 Agent：

```text
帮我创建一次 OME 复盘，目标是提炼一条可复用的编码经验：

1. 用 OME reflect 流程深度扫描可访问的编码会话来源，找真实的用户纠正，不要只看最后几条消息。
2. 提炼不超过 3 条经验草稿。
3. 只保留下次能改变 Agent 行动的经验。
4. 完成后只给我经验草稿审批和简短说明，不要让我看 JSON 或内部文件。
```

`--focus` 是扫描镜头，不是捷径。除非你明确限制来源，否则 Agent 仍然应该围绕这个
关注点深度检查可访问的相关会话来源。

## 2. 审批和补充

打开生成的经验草稿审批。重点看：

- 这个场景以后还会发生吗？
- Agent 下次看到这张卡，会改变行动吗？
- 它是否足够具体，能避免宽泛关键词误召回？
- 排除场景是否清楚？

你可以直接在对话里回复：

```text
第 1 条通过。
第 2 条太泛，按我刚才说的边界改一下。
这两条合并。
先别入库，我补一个反例。
```

## 3. 确认入库

确认无误后，告诉 Agent：

```text
确认入库通过的经验。
```

没有你的明确确认，经验不会进入后续自动召回。

## 4. 验证召回

用一个真实的未来任务验证，不要照抄卡片原文：

```bash
ome match "一个相似的未来任务" --explain
```

结果应该足够精确。如果卡片命中过宽，先改卡片，不要继续加更多卡。
