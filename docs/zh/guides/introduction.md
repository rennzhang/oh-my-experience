# 介绍

Oh My Experience 是一个本地工具，它记住你在 AI 编码中踩过的坑，在合适的时机自动
提醒 Agent。

你在真实工作中纠正 Agent 的错误。这些纠正沉淀为**经验卡**——一条行为修正加上
触发条件。下一次类似任务出现时，hook 自动把相关卡片注入 prompt。

不是在每轮对话里无脑堆规则。**在对的时刻，注入对的提醒。**

## AGENTS.md 不够吗？

AGENTS.md 和 CLAUDE.md 有两个问题。

### 常驻上下文膨胀

每轮对话都带着所有规则。但大部分规则是条件性的——只在特定任务里有用。

比如「UI 做完要跑真实浏览器验证」，改后端的时候根本用不上，却一直占着 token。
规则越多，每条的存在感越弱，Agent 反而更容易忽略真正该看的那条。

OME 把条件性规则从常驻上下文里拿出来。卡片只在 task envelope 匹配到触发条件时
才注入。

### 上下文压缩会丢掉规则

AGENTS.md 只在对话开头注入一次。长对话触发上下文压缩后，开头的规则可能就丢了
——Agent 做着做着突然忘了你定过的规矩。

OME 的 hook 每次 `UserPromptSubmit` 都跑一遍，经验每轮重新注入，不会被压缩吃掉。

### 两者怎么共存

不需要扔掉 AGENTS.md。它继续放那些在什么场景下都该生效的硬规则。OME 负责在合适
的时机补上条件性提醒。

## Memory 和 Skill 之间缺的那一层

Memory 记住事实。Skill 封装流程。**OME 记住执行判断。**

- **Memory** 可以记录你重视 UI 质量。
- **Skill** 可以知道怎么跑 Playwright。
- **经验卡片** 提醒 Agent：UI 任务没有经过真实浏览器、响应式状态、交互行为、
  加载状态、错误处理和控制台检查，就不能说完成。

这不是事实，不是工具——是上次翻车后沉淀下来的执行判断。

## 怎么运作

```
真实工作 → 复盘扫描 → 候选经验 → 你审核 → active 入索引
→ prompt 阶段自动召回 → 统计命中率 → 持续迭代
```

1. **导入会话**：从 Codex（或可选 Spool）导入历史记录
2. **复盘扫描**：Agent 检查会话，找到你纠正过的地方，生成候选经验卡
3. **你审核**：逐条 approve / reject / merge / rewrite
4. **卡片 active**：只有你明确批准后才进入 active
5. **自动召回**：hook 把每次新 prompt 匹配 active 卡片

未经你审核的卡片永远不会进入召回。AI 产生的噪声不会变成永久规则。

## 实际效果

**用之前：** AGENTS.md 写了 200 行规则。Agent 修 UI 时仍然跳过浏览器验证——规则
被淹没在噪声里。你又纠正了一遍。

**用之后：** AGENTS.md 只剩 30 行硬规则。你说「修一下登录页 UI」，hook 检测到 UI
任务，召回一条经验卡：「UI 任务必须先过真实浏览器验证才能说完成」。就这一段
额外上下文。Agent 在报告完成前主动跑了一遍浏览器。

## 安全设计

- **热路径不调 LLM。** 确定性 BM25-like 匹配，不需要 API Key，不走网络。
- **Fail-open。** Hook 出错就返回空上下文，不阻塞 Agent。
- **不存原始 prompt。** Hook 日志默认只存 prompt hash + 任务摘要。原文记录 opt-in 且带 TTL。
- **不过审不进库。** 没有任何自动化流程能绕过人工审核直接写 active。
- **全在本机。** 不依赖云服务、不连数据库。
