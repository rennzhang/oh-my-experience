# 复盘与草稿审批

OME 的核心价值在于：把你纠正 Agent 的真实经验，沉淀为可复用的经验。

这个过程叫 reflect（复盘）。用户只需要做三件事：让 Agent 复盘、确认经验草稿、
决定哪些入库。内部的 JSON、编号、草稿和启用命令都应该由 Agent 处理。

```text
真实工作 → Agent 复盘 → 经验草稿审批 → 继续提意见优化 → 确认入库 → 后续自动召回
```

## 什么时候做复盘

不需要每次对话都做。适合的时机：

- 你纠正了 Agent 2-3 次同类错误
- Agent 又犯了一个你已经纠正过的错（说明该入库了）
- 完成一个重要阶段，想做一次回顾
- `ome stats` 显示很多 prompt 没有命中任何经验（说明缺卡）

## 做一次复盘

### 步骤 1：发起复盘

把这段话复制给 Agent：

```text
帮我做一次 OME 经验复盘，关注最近我纠正过你的执行错误。

要求：
1. 你自己完成 OME reflect 流程和来源审计，不要让我看 JSON 或内部文件。
2. 只提炼以后还会复用的经验，一次最多给 5 条。
3. 完成后给我经验草稿审批链接和简短说明，等我确认。
4. 如果我继续补充想法、反例或修改意见，就优化同一次复盘，不要另开一轮。
5. 等我明确说“确认入库”后，再把通过的经验入库。
```

### 指定关注方向

如果你已经有明确想沉淀的经验，加 `--focus`：

```bash
ome reflect start --focus "浏览器验证和交付前检查"
ome reflect start --focus "Git 提交规范和 PR 流程"
ome reflect start --focus "TypeScript 类型安全和错误处理"
```

`--focus` 不是过滤条件，而是告诉 Agent 扫描时的侧重点。Agent 还是会检查所有
可访问的会话来源，只是聚焦分析指定方向。

### 复盘出来的经验长什么样

每条经验只需要让你判断四件事：

- 这条经验在说什么。
- 什么场景下应该触发。
- 什么相近场景下不该触发。
- 未来 Agent 真正会执行哪条规则。

好的经验是具体的行为修正。太泛的“重视代码质量”、只针对上一次某个文件的细节、
没有可执行动作的总结，都不应该入库。

### 步骤 2：审批和继续优化

Agent 展示经验草稿审批后，你不用记命令，直接在对话里说你的决定或想法：

- “第 1 条通过。”
- “第 2 条太泛，改得更像可执行规则。”
- “这两条可以合并。”
- “先别入库，我再补一个反例。”
- “按我刚才说的优化同一次复盘。”

审批前问自己三个问题：
1. 这条经验以后还会遇到吗？
2. 下次 Agent 看到它，能避免犯同样的错吗？
3. 使用标准和忽略标准够精确吗？会不会误触发？

### 继续修订同一次扫描

经验已经展示后，如果你继续给 Agent 一个链接、一段粘贴内容、一句纠正，或者说
“这个可以参考吸收”，默认意思是优化当前这次复盘，不是另开一轮扫描，也不是新增
一张并列经验。

除非你明确要求另起主题或单独做一条经验，否则经验草稿审批应该继续停留在同一次复盘，直到
你确认入库或放弃。

### 步骤 3：入库

审批和优化完成后，直接告诉 Agent：

```text
确认入库这次复盘里通过的经验。
```

Agent 应该先按审批意见准备入库，再启用通过的经验。没有你的明确确认，经验不会进入
后续自动召回。

### 步骤 4：验证

入库完成后立即验证：

```text
用 `ome match` 验证刚入库的经验卡。
模拟一段跟刚才翻车场景类似的任务描述，看新卡会不会命中。

ome match "一段任务描述" --explain
```

## 常用回复

| 你想做什么 | 可以直接说 |
|------|------|
| 这条经验可以入库 | `第 1 条通过。` |
| 这条不要 | `第 2 条不通过，太泛了。` |
| 两条表达同一件事 | `把第 1 条和第 3 条合并。` |
| 方向对但不够准 | `第 2 条按这个边界改一下：...` |
| 还要继续补材料 | `先别入库，我再补一个反例。` |

## 保持经验库健康

经验库不是越多越好。几条长期建议：

- **宁可少而精。** 10 条精准召回的经验卡，比 50 条模糊规则更有用。
- **定期看 stats。** `ome stats` 看哪些卡长期没命中，考虑归档。
- **查重。** 如果新经验和旧经验差不多，先合并再入库。
- **修正误触发。** 如果某张卡经常在不相关的任务中被召回，收紧 `criteria.ignore_when`、`recall.triggers`、topics 或 scope。

## 给 Agent 的一键复盘 prompt（模板）

以后每次想做复盘，直接用这段：

```text
帮我对最近 [时间段 / 项目名] 的编码会话做一次 OME reflect 复盘扫描。

要求：
1. 你自己完成 OME reflect 流程、来源审计和内部候选写入。
2. 浏览会话中我纠正过你的地方，提炼 ≤5 条经验草稿。
3. 完成后只给我经验草稿审批链接和简短说明，不要让我看 JSON、内部文件或候选 schema。
4. 如果我继续补充想法、反例或修改意见，就优化同一次复盘，不要另开一轮。
5. 等我明确说“确认入库”后，再把通过的经验入库。

只提取真正能复用的执行判断，不要把一次性上下文写成经验。
```
